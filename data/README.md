# Pal data

This directory holds the bundled Pal database. The web app **never fetches at runtime** ‚Äî JSON here is what ships in the build.

## Files

- `pals.json` ‚Äî array of `Pal` records, one per Pal.
- `combos.json` ‚Äî array of `BreedingCombo` records (special-breeding overrides).
- `passives.json` ‚Äî master list of `PassiveSkill` records (game-wide passive catalog).
- `reverse-index.json` ‚Äî generated by `pnpm build:reverse-index`. Maps `childId ‚Üí [{parentA, parentB}, ‚Ä¶]`. **Do not hand-edit** ‚Äî re-run the builder.

## Phase 1 status

The current `pals.json` is a **hand-curated placeholder** of ~25 Pals, used so the calculator runs end to end before the scraper is wired up. The `powerValue` numbers are approximate community values and should be considered _provisional_ until verified against a single canonical source.

The hand-curated set is intentionally small. Once the scraper is pointed at a real source, it should overwrite this file in full.

## Schema

### `Pal`

```ts
{
  id: string;            // stable lowercase id, e.g. "lamball"
  paldexNo: number;      // Paldex number
  name: string;          // display name
  slug: string;          // url-safe identifier
  elements: Element[];   // 1‚Äì2 elements, primary first
  powerValue: number;    // CombiRank ‚Äî used by breeding math
  breedable: boolean;    // false for variants / special-only Pals
  imageUrl?: string;     // local "/pals/lamball.png" or remote
  passiveTraits?: string[];          // reserved for Phase 3
  workSuitabilities?: Record<string, number>; // reserved for Phase 4
}
```

`Element` is one of: `Neutral`, `Fire`, `Water`, `Grass`, `Electric`, `Ice`, `Ground`, `Dark`, `Dragon`.

### `BreedingCombo`

```ts
{
  parentA: string;  // Pal id
  parentB: string;  // Pal id
  child: string;    // Pal id (overrides power-value math)
}
```

Order does not matter ‚Äî the engine looks up `(A, B)` and `(B, A)` symmetrically.

### `PassiveSkill`

```ts
{
  id: string;       // "swift", "lucky", "ferocious"
  name: string;     // display name
  tier: "positive" | "negative" | "neutral";
  rank: 1 | 2 | 3 | 4;  // for stat-boosting passives; 1 for non-tiered
  effect: string;   // human-readable description
}
```

Passives are mostly per-instance random rolls in Palworld ‚Äî they're not fixed to specific Pals. This master list exists so the UI can autocomplete, color, and describe passives, and so the inheritance math knows the global pool size.

### Passive inheritance constants

The constants in `src/lib/breeding/passives.ts` (`countDistribution`, `wildPassiveChance`) are **community estimates**, not official Pocketpair numbers. Update them when better consensus lands; the rest of the calculator will follow automatically.

## Recommended scraping sources

These are the candidate community sources. **Pick one** and set `PAL_DATA_SOURCE` in `.env` before running the scraper. Review terms of service and `robots.txt` before scraping.

- **paldb.cc** ‚Äî well-structured, machine-readable. Good first choice.
- **palworld.gg** ‚Äî community wiki with breeding data.
- **palworld.fandom.com** ‚Äî Fandom wiki, broad coverage but inconsistent formatting.

The scraper is source-agnostic and reads the base URL from `PAL_DATA_SOURCE`. Adapt `scripts/lib/parsers.ts` to the chosen source's HTML/JSON shape.

## Regenerating

```sh
pnpm scrape           # runs scrape-pals + scrape-combos
pnpm scrape:pals      # just pals
pnpm scrape:combos    # just combos
```

The scraper writes back into this directory and caches HTTP responses under `.cache/` (gitignored). Commit the resulting `pals.json` / `combos.json` when you're satisfied with the output.
